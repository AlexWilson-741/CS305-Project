# Artemis Financial Security Project

This is a mock security project wherein Artemis Financial, a financial service company, required improvements to its application security, specifically concerning protection of client data and ensuring secure communication. The main issue was the lack of data integrity verification and encrypted communication between the client and server. I addressed this by implementing SHA-256 hashing to generate checksums for verifying data integrity and configuring HTTPS using a self-signed SSL certificate to secure data in transit. I also used dependency-check to identify existing vulnerabilities and verified that my refactored code did not introduce any new ones. Coding securely is important because it protects sensitive information, prevents tampering, and reduces the risk of financial loss and reputational damage, which is especially important in the financial sector.

One of the more challenging aspects was correctly implementing and understanding cryptographic concepts like hashing and secure communication, but this was also the most valuable part of the project. I improved security by layering protections, including encryption, hashing, and vulnerability scanning, and validated these changes through functional testing and static analysis. I ensured the application remained functional by testing the /hash endpoint with multiple inputs and confirming consistent behavior, while also reviewing the code to avoid introducing new vulnerabilities. Tools such HTTPS configuration and OWASP Dependency-Check were used throughout the process. This project demonstrates my ability to identify vulnerabilities and validate software security, which makes it a great example to present to future employers.
